## 1. コサイン類似度って何？
**「2つのものが、どれくらい同じ方向を向いているか」を測る方法**だよ。  

名前はむずかしそうだけど、イメージは**矢印**。

- あるデータを「数字の並び」で表す
- その数字の並びを、数学では**ベクトル（矢印）**として見る
- 2本の矢印の**向きが近いほど似ている**と考える

### たとえば
好きな科目を数字で表すとする：

- Aさん = `[数学5, 英語1, 体育1]`
- Bさん = `[数学4, 英語1, 体育1]`
- Cさん = `[数学1, 英語5, 体育1]`

AさんとBさんは似てそう。  
AさんとCさんはあまり似てなさそう。  

この「似てる度」を数字で出すのが**コサイン類似度**。

---

## 2. なんで「コサイン」なの？
2本の矢印の間の**角度**を見るから。  

- 角度が **小さい** → 向きがほぼ同じ → **似ている**
- 角度が **大きい** → 向きが違う → **似ていない**

コサイン類似度は、ざっくりこう覚えればOK：

- **1に近い** → かなり似てる
- **0に近い** → あまり関係ない
- **-1に近い** → 真逆

高校生向けには、まず  
**「1に近いほど似ている」**  
だけ覚えれば十分。

---

## 3. たとえ話
### YouTubeやNetflixのおすすめ
動画や映画を、特徴の数字で表すとする。

例：
- アクション度
- 恋愛度
- コメディ度
- 勉強になる度

ある映画Aと映画Bが、こういう特徴で**同じようなバランス**なら、コサイン類似度は高くなる。  
だから「これ見た人はこれも好きそう」が作れる。

---

## 4. コサイン類似度のポイント
### 長さより「方向」を見る
たとえば：

- A = `[1, 2, 3]`
- B = `[10, 20, 30]`

BはAを10倍しただけ。  
大きさは違うけど、**向きは同じ**。  

だからコサイン類似度では  
**AとBはすごく似ている**  
と判定される。

これは、
- テストの点数の絶対値より
- **得意・苦手の傾向が似てるか**
を見たいときに便利。

# 5. 近傍探索法って何？
これは  
**「たくさんあるデータの中から、似ているものを探す方法」**  
のこと。

「近傍（きんぼう）」は、簡単に言うと**近くにある仲間**。

つまり近傍探索法は、

- ある1つのデータを決めて
- データベースの中から
- **それに近いものを探す**

という方法。

---

## 6. たとえ話：クラスで似た人探し

- 数学が好きか
- 英語が好きか
- スポーツが好きか
- 音楽が好きか

を数字にしたとする。  

すると1人1人を「数字のベクトル」で表せる。  

このとき、
**Aさんに似ている人を探す**  
のが近傍探索。

- Aさんと全員を比べる
- コサイン類似度を計算する
- 数字が高い順に並べる

これで  
**Aさんの近くにいる人（近い特徴の人）**  
が見つかる。

## まず前提
k-NN = **近い順に上位 k 件を探す**

OpenSearch の k-NN は、実質 **ベクトル検索をクラスタ全体で回せる仕組み**。

---

## 「分散ANNに強い」の意味
OpenSearch はもともと検索エンジンで、データを

- **シャード** に分割して
- **複数ノード** に配置して
- 並列に検索する

のが得意。  
その仕組みに **ベクトル近傍探索（ANN）** を乗せられるのが強み。

つまり：

1. ベクトルデータが大量にある  
2. 1台では載らないので複数台に分ける  
3. 各ノード・各シャードで ANN を実行する  
4. 最後に結果を集めて全体の上位 k 件を返す  

この流れを自然にできる。

---

## イメージ
1000万件のベクトルがあるとして、  
それを 10 台に 100万件ずつ置く。

クエリが来たら、

- 1台目で「近い候補」を探す
- 2台目でも探す
- …
- 10台目でも探す
- 最後に全部の候補をまとめて、全体トップ k を決める

これが **分散 ANN** の基本イメージ。

OpenSearch はベクトルだけじゃなく、

- キーワード検索

- テキスト relevance
- 集計

これが実運用でかなり大きい。

「似ていて、かつ日本語で、かつ公開中で、かつ特定カテゴリ」

### 4. 可用性・運用の仕組みがある
レプリカ、再配置、監視など、  
もともと分散検索エンジンとしての仕組みがあるので、
