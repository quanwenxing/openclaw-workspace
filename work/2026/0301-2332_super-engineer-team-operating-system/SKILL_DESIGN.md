# SKILL_DESIGN.md

## 方針
- できる限り作業をスキル化し、必要時にエージェントが呼び出す。
- スキル本文は短く、核心に集中する（冗長説明を避ける）。
- 各スキルは「実行手順」「品質ゲート」「成果物形式」を持つ。

## スキル最小テンプレ
```md
---
name: <skill-name>
description: <何をするか + いつ使うか>
---

1. ゴールを確認し、受入条件を明文化する。
2. 最短で実装し、必須テストを自動実行する。
3. 失敗時は原因を特定し、自己修正ループで再検証する。
4. 変更点・証跡・残リスクを出力して完了する。
```

## スキル分類（初期）
- `spec-writer`: 要件化
- `task-planner`: タスク分解/順序最適化
- `feature-implementer`: 実装
- `qa-runner`: テスト実行とレポート
- `release-operator`: push/deploy

## 呼び出し規約
- 1タスク1主スキル（必要に応じ補助スキル）
- 実行前に入力不足があれば、推測せず最小質問のみ
- 実行後は必ずレポートを残す
- 実行中は Self-Directed Execution Loop（Goal→現状評価→次ステップ決定→実行）を維持する
- ゴール達成まで自律継続し、「次何するか」を都度ユーザーに聞かない
