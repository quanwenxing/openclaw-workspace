# TEST_POLICY.md (v1)

## 1. 目的
実装エージェントが自律的にテストを実行し、環境差異があっても品質を担保する。

## 2. テスト実行優先順位
1. Chrome MCP（第一優先）
2. Playwright（フォールバック1）
3. curl（フォールバック2）

## 3. 可用性チェック
- テスト開始時に Chrome MCP の接続確認を1回実施
- 接続不可/操作開始不可なら即 Playwright へ切替

## 4. フォールバック条件
### Chrome MCP → Playwright
- 接続不可: 即時切替
- 操作中エラー: 2回リトライ後に切替
- タイムアウト: 15秒で切替

### Playwright → curl
- 起動不可: 即時切替
- 主要シナリオ失敗: curl最小テストへ移行
- タイムアウト: 30秒で切替

### curl
- エンドポイントごとに10秒タイムアウト
- 最低限の疎通・認証・主要API正常系を確認

## 5. 共通テスト要件
- バグ修正時は再発防止テストを1件以上追加
- PR前に自動テスト一式を実行
- 失敗時は自己修正ループ（最大N回、初期値3）

## 6. レポート要件（必須）
`TEST_REPORT.md` に以下を記録:
- 使用経路（Chrome MCP / Playwright / curl）
- フォールバック理由
- リトライ回数
- 失敗内容と修正内容
- 未解決リスク

## 7. マージゲート
- 重大テスト失敗 0
- 新規機能に対応するテスト追加済み
- CHANGELOG/RESULT更新済み
