# Coding Agent Collaboration Skill (claude-codex-collab) - Revised Plan

## 連携構成（オーケストラ型）
OpenClaw（クロウ）をハブとし、2つのCLIを以下の役割で制御する。

| エージェント | 役割 | 実行モード |
| :--- | :--- | :--- |
| **Claude Code** | 実装担当 (Builder) | `--print` による自動実行（非対話） |
| **Codex CLI** | レビュー担当 (Reviewer) | 非対話モードでのコード解析・評価 |
| **Claw** | 司令塔 (Orchestrator) | 指示の作成、出力のパース、修正依頼のループ管理 |

## 実現できること
- **自律的品質保証**: 実装したコードを別視点（別モデル）で即座にレビューし、脆弱性やバグを未然に防ぐ。
- **マルチモデル思考**: Claudeのコーディング能力と、Codex（GPT系等）の論理チェック能力を掛け合わせる。
- **自動修正ループ**: 「レビュー指摘 → 修正 → 再レビュー」を人間が介在せずに完了させる。

## 再検討による考慮漏れと対策
- **1. プロンプトの断絶**: 
    - 懸念：Claudeが書いた背景（何をしようとしていたか）がCodexに伝わらない。
    - 対策：クロウがClaudeの履歴や実行ログを要約し、Codexに「背景情報」として渡す。
- **2. 修正無限ループの防止**:
    - 懸念：些細なスタイルの違いで二人が言い合いになり、終わらなくなる。
    - 対策：最大ループ回数（デフォルト3回）の設定と、クロウによる「この程度ならOK」という最終判断フェーズの導入。
- **3. ファイル競合と一貫性**:
    - 懸念：二人が同時にファイルを触ったり、古いコードをレビューしたりする。
    - 対策：作業ディレクトリを厳格に管理し、レビュー対象は「現在のGit diff」や「特定のスナップショット」に限定する。
- **4. コストとトークン管理**:
    - 懸念：コンテキストが長くなり、トークン代が跳ね上がる。
    - 対策：成果物以外の不要なログを削ぎ落とし、必要なファイルのみを各エージェントに提示する「コンテキストのつまみ食い」制御。
- **5. 人間の介入ポイント**:
    - 懸念：全自動すぎて、間違った方向に進んでいる時に止められない。
    - 対策：各フェーズの合間にOpenClawが「今の状況」を報告し、もんちゃんが `continue` か `stop` か選べるチェックポイントを設ける。

## 構成案
- `SKILL.md`: 連携フロー、エージェントへの役割指示プロンプトの定義。
- `scripts/agent-wrapper.sh`: 各CLIのパス解決、タイムアウト制御、出力キャプチャ。
- `scripts/orchestrator.py`: ループ管理、パース、プロンプトの動的生成（Pythonでロジックを組む）。
