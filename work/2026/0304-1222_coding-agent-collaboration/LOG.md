# 2026-03-04 12:26
- 今回の設計は、OpenClaw（クロウ）が「オーケストレーション」を担当する形。
- 仕組み：
    - クロウが Claude Code を「開発担当」として起動し、指示を出す。
    - 成果物を取得・解析し、それを Codex に「レビュー担当」として渡す。
    - Codex の指摘をまとめ、Claude Code に再指示。
- メリット：
    - ユーザーはクロウに「これを二人でやって」と頼むだけで済む。
    - エージェント間のコンテキストの受け渡し（プロンプトの調整）をクロウが最適化できる。
    - CLI が入っている環境ならどこでも（tmux なしでも）動く。
