# API利用の可否と代替案

## 1. 競合アプリ公式APIの利用
- **可否**: 否（プライベートAPIのため）。
- **リスク**: 
    - 認証キーの抽出（リバースエンジニアリング）が必要で、利用規約違反となる。
    - 仕様変更により、突然プログラムが動かなくなるリスクが高い。

## 2. 代替となる強力な公開API
- **Google Gemini 1.5 Flash / Pro** (推奨): 
    - 画像認識と構造化抽出が極めて強力。
    - 個人開発なら「無料枠」でも十分に試作可能。
- **Google Document AI**: 
    - ドキュメント解析に特化。高精度なレイアウト解析が必要な場合に有効。
- **OpenAI GPT-4o with Vision**: 
    - 汎用性が高いが、Gemini 1.5の方がコンテキストウィンドウや処理速度で有利な面がある。

## 3. 週末プロジェクトの戦略
「既製品のAPI（Gemini等）を使い、自分流のPDF生成ロジック（ReportLab等）を組み合わせる」ことで、競合アプリを超えるカスタマイズ性を持たせるのが、最も効率的な勝ち筋。
