Dexter って何？ virattt/dexter

一言でいうと： 「金融リサーチ専用の自律エージェント」です。質問を受けると、 調査計画を立てる → データを集める → 自分で検証する を回して、 根拠付きの回答を作る設計です。

READMEにある表現を噛み砕くと、Claude Codeの金融リサーチ特化版という位置づけです。

このリポジトリで「できること」

1) 金融質問を分解して調査する

  • 複雑な質問を小さな調査タスクに分割
  • 順番に実行し、途中で方針修正
  • 最終的にまとまった回答を返す

2) 財務データを使える

  • 損益計算書（Income Statement）
  • 貸借対照表（Balance Sheet）
  • キャッシュフロー計算書（Cash Flow）

3) 自己チェック機能

  • 自分の作業を検証しながら進む
  • 不十分なら反復して改善
  • 暴走防止（ループ検知 / ステップ上限）あり

4) 実験・評価しやすい

  • 評価スイートあり（正答率などを確認）
  • LangSmith連携でトレース可視化
  • ツール呼び出しログをJSONLで保存

技術スタック（ざっくり）

  • 実行基盤：Bun + TypeScript
  • UI：Ink（ターミナルUI）
  • LLM接続：OpenAI / Anthropic / Google / Ollama など
  • 検索：Exa（優先）+ Tavily（フォールバック）
  • ブラウザ系：Playwright（postinstallでchromium導入）

使い始める流れ

git clone https://github.com/virattt/dexter.git
cd dexter
bun install
cp env.example .env
# .env に API キーを設定
bun start

開発中は bun dev、評価は bun run src/evals/run.ts。

向いている用途 / 向いていない用途

向いている

  • 企業分析の一次ドラフト
  • 複数指標の横断調査
  • 「調査プロセスを残したい」ケース

注意点

  • 投資助言そのものではない（最終判断は人間）
  • APIキー準備が前提
  • データソース品質に結果が依存
