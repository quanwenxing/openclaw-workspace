# tts-on-demand

## 目的
ユーザー体験を優先して、必要なときだけ文字を音声に変換する。

## 実行ルール
1. 次のどちらかを満たすときだけTTS化する。
   - ユーザーの明示指示（例: 「TTSで」「音声にして」「読んで」）
   - アシスタント判断で音声のほうが適切（短い挨拶・読み上げ依頼文脈・ハンズフリー想定など）
2. 明示指示がない通常会話はテキスト優先。
3. 検証時は勝手にメッセージ送信しない。先に結果を報告し、送信依頼があれば送る。
4. 1ユーザー要求につき原則1回生成。重複生成しない。

## 手順
1. `tts` ツールで音声を生成する（channelは会話チャネルに合わせる）。
2. 生成結果の `MEDIA:` パスを返す。
3. 実送信依頼がある場合のみ `message` ツールで送る。

## 品質・失敗時対応
- 生成ファイルが0バイト、または送信で `file must be non-empty` が出たら失敗扱い。
- 失敗時は次を実行する。
  1) 同文で1回だけ再生成
  2) それでも失敗ならテキストで状況を報告し、再試行可否を確認
- 検証で作成した一時音声は不要なら削除する。

## 返答方針
- ユーザー向け文は短く要点だけ。
- 音声生成後は `MEDIA:` をそのまま返す（TTSレスポンス形式を崩さない）。
- 実送信を行った場合は重複返信を避ける。
